#### 和SSR的区别
完全不是一个东东：

1. SSR是在服务端把js转成HTML，返回给客户端；而Server Components是在客户端渲染的，服务端输出的是chunks

2. SSR的请求响应是HTML，无法保留客户端的状态，而Server Components在渲染的时候，是不影响Client Components的状态的

实际上SSR和Server Components可以结合使用，SSR目前大部分被用在首屏加载场景，在服务端把js转成html，返回给浏览器渲染，之后的页面交互可以选择用Server Components。nextJS 就对此进行了整合，实现更好的性能。

## \<Suspense\>
[\<Suspense\> 官方文档](https://zh-hans.react.dev/reference/react/Suspense#suspense)

#### 注意
只有启用了 Suspense 的数据源才会激活 Suspense 组件，它们包括：
* 支持 Suspense 的框架如 Relay 和 Next.js。
* 使用 lazy 懒加载组件代码。
* 使用 use 读取缓存的 Promise 值。

Suspense 无法 检测在 Effect 或事件处理程序中获取数据的情况。

在上面的 Albums 组件中，正确的数据加载方法取决于你使用的框架。如果你使用了支持 Suspense 的框架，你会在其数据获取文档中找到详细信息。

目前尚不支持在不使用固定框架的情况下进行启用 Suspense 的数据获取。实现支持 Suspense 数据源的要求是不稳定的，也没有文档。React 将在未来的版本中发布官方 API，用于与 Suspense 集成数据源。

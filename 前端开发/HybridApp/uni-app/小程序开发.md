* 安装 HBuilderX 后，还要安装”微信开发者工具“。微信开发者工具中需要在 设置-安全设置 中开启服务端口。
* 小程序不支持在css中使用本地文件，包括本地的背景图和字体文件。需以base64方式方可使用。但是当使用的背景图小于40k时，uni-app 会自动转成 base64 格式，所以可以直接引用。如果背景图大于40k，可以手动转成 base64 或使用服务器上的图片。
* 尽量使用 flex 布局，兼容性更好。
* 需要配置好小程序的 appId 并将自己的微信号加入到开发者中才能进行预览、真机调试、上传等功能。
* 运行环境：在 uni-app 中点击“运行”编译出来的代码是开发环境，点击“发行”编译出来的代码是生产环境。还可以通过编辑 package.json 来自定义环境。
    ##### package.json 添加测试环境
    ```json
    "uni-app": {
    	"scripts": {
    		"mp-wx-test": {
    			"title": "小程序测试环境",
    			"env": {
    				"UNI_PLATFORM": "mp-weixin",
    				"NODE_ENV": "定义 NODE_ENV 没用，内置的优先级更高",
    				"TEST": "true"
    			},
    			"define": {
    				"WX-TEST": true
    			}
    		}
    	}
    }
    ```
    ##### 环境判断
    ```js
    const isDev = process.env.NODE_ENV === 'development';
    const isTest = !!process.env.TEST;
    export const ENV = isTest ? 'test' : (isDev ? 'dev' : 'prod');
    
    const hosts = {
    	dev: 'http://dev.com',
    	test: 'http://test.com',
    	prod: 'https://prod.com'
    };
    export const host = hosts[ENV];
    ```


## 存在的问题：
* 跳转时注意路径，pages 目录中的页面使用 uni.redirectTo 等方法跳转时可以使用相对路径，其它页面最好使用绝对路径，如 /pages/login/login，并且注意最后的 login 不能写成 login.vue。比如自己新建了个 common/utils.js 文件，里面使用跳转，如果用相对路径也会出现找不到页面的情况。如果跳转失败，可以在 fial 方法中打印接收的参数查看具体原因。
* 启动时立刻进行跳转会出现首页始终要显示的情况。比如需要启动时判断登录，默认加载首页，没登录就直接重定向到登录页。uni-app 的首页就始终要出现后再跳转。考虑使用启动动画提升体验。。。
* uni-file-picker 删除数据时，组件会修改绑定的 model 数据，自己改了下源码不让组件修改，通过触发的 delete 方法自行修改。
* 使用体验版等出现网络不能正常请求的情况，可尝试开启调试模式。比如测试的时候使用的局域网 IP 地址，而局域网 IP 无法配置到小程序服务器列表中，但是小程序是支持局域网访问的，只是需要开启调试模式才能访问。

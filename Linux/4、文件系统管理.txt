--分区类型：是硬盘限制，与系统无关
主分区：最多只能分4个。
扩展分区：1、只能有一个。2、也算作主分区的一种，主分区+扩展分区最多只能有4个。3、不能存储数据和格式化，只能包含逻辑分区。
逻辑分区：在扩展分区中划分，最多支持59个逻辑分区。
分区号1，2，3，4只能由主分区使用，逻辑分区都是从5开始。

格式化：
主要作用并不是清空数据，清空数据只是其中附带的作用。
主要作用是根据用户选定的文件系统（如：FAT32、NTFS、EXT4）往磁盘里写入特定数据。在分区中划出一片用于存放文件分配表、目录表等用于文件管理的磁盘空间。

Linux安装需要的分区：
/  根分区
swap  分区  类似 Windows 的虚拟内存，1-2GB
/boot  分区  启动分区，200MB
Linux 的分区和 Windows 不同，如 /boot 分区，是在 / 根分区下的，而不像 Windows 每个分区是平级的。也就是说，Linux 的分区是将某个目录指向分区
系统常用目录：
/root  系统管理员的用户目录
/home  其它用户的目录，比如新建了个 Tirion 用户，就会有 /home/Tirion 这个目录
/mnt  挂在目录，如挂载U盘等
/dev  设备目录。Linux 中，硬件都可以看做是个设备。
    SATA硬盘 /dev/sda1（sd 表示 SATA 硬盘，a 表示第一块硬盘，1表示第一个主分区。而划分的逻辑分区会从编号5开始）
    光驱 /dev/cdrom 或 /dev/hdc
    鼠标 /dev/mouse
    ...
/etc  系统管理和配置文件
/lib  标准程序设计库
/bin /sbin  二进制目录，里面都是一些可执行的程序和命令
/var  放置系统执行过程中经常变化的文件，如随时更改的日志文件
/lost+found  这个目录平时是空的，系统非正常关机而留下“无家可归”的文件（windows下叫什么.chk）就在这里
/tmp  公共的临时文件存储点
/usr  Unix software resource。应用程序存放目录，/usr/bin 存放应用程序， /usr/share 存放共享数据，/usr/lib 存放不能直接运行的，却是许多程序运行所必需的一些函数库文件，/usr/local 存放软件升级包，/usr/share/doc 系统说明文件存放目录。
/proc  虚拟文件系统，该目录中的数据并不保存到硬盘中，而是保存在内存中。保存 CPU 信息、设备信息、网络信息等。
/sys  和 proc 类似，主要保存内核相关信息。

文件命名规则：
除了/之外的所有字符都可以作为文件名，但不建议使用特殊符号，另外大小写敏感。
.开头的文件表示隐藏文件

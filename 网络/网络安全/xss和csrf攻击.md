# XSS 攻击
XSS 中文名叫跨站脚本攻击。就是用户在页面上的输入框中输入 \<script\> 脚本，从而达到攻击的目的。XSS 攻击其实和 SQL 注入是类似的，只是 SQL 注入是后端的问题，而 XSS 是前端的问题。

前端在接收用户输入的时候就需要对输入内容进行编码，而不是直接提交给后端存储。

# CSRF 攻击
CSRF 中文名叫跨站请求伪造。比如我在可信的 A 网站登录后，然后又打开了恶意网站 B，而 B 网站会发送恶意请求给 A 网站，这时这个恶意请求就会带上 **A 网站**的 cookie 从而请求成功。

解决办法：不使用 cookie，使用 token 进行用户验证就行了。如果一定要使用 cookie 那可以在请求中加上一些用于验证的其它字段，而不是单纯的使用 cookie 进行验证。

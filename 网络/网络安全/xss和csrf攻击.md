# XSS 攻击
XSS 中文名叫跨站脚本攻击。就是用户在页面上的输入框中输入 \<script\> 脚本，从而达到攻击的目的。XSS 攻击其实和 SQL 注入是类似的，只是 SQL 注入是后端的问题，而 XSS 是前端的问题。

前端在接收用户输入的时候就需要对输入内容进行编码，而不是直接提交给后端存储。

# CSRF 攻击
CSRF 中文名叫跨站请求伪造。比如我在可信的 A 网站登录后，然后又打开了恶意网站 B，而 B 网站会发送恶意请求给 A 网站，这时这个恶意请求就会带上 **A 网站**的 cookie 从而请求成功。

解决办法：不使用 cookie，使用 token 进行用户验证就行了。如果一定要使用 cookie 那可以在请求中加上一些用于验证的其它字段，而不是单纯的使用 cookie 进行验证。

CSRF 攻击流程：
1. 用户C打开浏览器，访问受信任网站A，输入用户名和密码请求登录网站A；

2. 在用户信息通过验证后，网站A产生Cookie信息并返回给浏览器，此时用户登录网站A成功，可以正常发送请求到网站A；

3. 用户未退出网站A之前，在同一浏览器中，打开一个TAB页访问网站B；

4. 网站B接收到用户请求后，返回一些攻击性代码，并发出一个请求要求访问第三方站点A；

5. 浏览器在接收到这些攻击性代码后，根据网站B的请求，在用户不知情的情况下携带Cookie信息，向网站A发出请求。网站A并不知道该请求其实是由B发起的，所以会根据用户C的Cookie信息以C的权限处理该请求，导致来自网站B的恶意代码被执行。

这里攻击主要发生在第5点，比如 B 网站有个按钮，点击后会使用 A 网站的登录信息（cookie）向 A 网站发起一个转账请求，如果 A 网站具有安全问题，就会导致转账成功。
**cookie 会在每次向该网站发起请求时由浏览器自动添加到请求头中。**

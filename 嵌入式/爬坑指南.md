### HAL_Delay 无效
必须先调用 HAL_Init()，否则调用 HAL_Delay 无效。

### GPIO 时钟使能
一般 MCU 会默认开启 GPIOA 的时钟使能，但是其它的使能可能需要手动开启，比如我在 py32 中设置引脚的输出，GPIOA 和 GPIOB 都能工作，但是 GPIOF 却不能正常工作。后面排查了很久才发现 GPIOF 需要开启时钟使能才行。
```c
RCC->IOPENR |= RCC_IOPENR_GPIOFEN;
// 或
__HAL_RCC_GPIOF_CLK_ENABLE();
```

### 中断中调用 delay
如在按键中断中调用 HAL_Delay 函数来消抖，可能会出现卡死，原因是按键中断的优先级可能比时钟滴答定时器的优先级更高。
在按键中断中调用 HAL_Delay，而 HAL_Delay 底层是通过滴答定时器产生的中断来计时的，当按键中断优先级比滴答定时器优先级高时，由于程序在执行按键中断，便无法触发优先级更低的滴答定时器中断，所以 HAL_Delay 也就无法正常计时，从而程序就卡死了。

# 报错
### invalid multibyte character sequence
从别的文件复制代码后报这个错，原因是另一个文件的文件编码和当前文件的编码不同。要么统一编码，要么删除警告部分手动输入。

//可以直接在标签中写JS代码 JS解析器： 1、先将JS代码全盘读入，在里面先找出一些需要的东西（预解析|预编译），如var function等。找到var声明的变量会先解析，但是值都是undefined；找到的函数也会解析，值就是函数的内容，这就是为什么函数声明的位置不重要（函数声明提升，意思是在执行代码之前会先读取函数声明）。遇到变量和函数重名，会保留函数，与顺序无关 （var a = 2; 在编译器看来是 var a 和 a = 2，而 var a 会提升到顶部，当代码运行到 var a = 2 这里时，其实只是运行 a = 2，这就是变量提升） 2、逐行解析代码，并运行。遇到表达式会修改保存的变量的值如：var a=3，a=a+2，a--，a=Number(a)，参数等能修改一个变量名的值的就是表达式，function a()是函数申明不是表达式。所以在运行代码过程中先声明了一个变量并赋值，然后又申明了一个同名的函数，变量的值会是赋予的值而不是函数，而这个函数也不会保存在代码仓库中，所以a()无法调用。 因此可以得出，在预解析过程中函数占有优先权，而在解析运行过程中变量声明占有优先权。所以切忌将变量和函数同名（在同一个域中） 域：每个是一个域（全局域），在一个文件中的不同属于不同的域，但是使用的是同一个仓库。所以引入的外部文件需要写在前面，这样在解析的时候才会先把那些变量保存在仓库中，之后才能使用；如果写在后面，由于是不同的那些引入的代码不在同一个域中，即使是函数也无法正常调用。每个函数内部也会开辟一个新的域（局部域），当调用函数的时候，局部域也会按照全局域相同的模式运行：预解析再解析执行。 例： 1、 2、 3、 中间还有一些其它步骤，比如语法分析等不多做介绍 不要在if或for循环里定义变量或函数，因为FF无法预解析，而别的浏览器可以 使用CDN优化可以加快用户访问速度，比如加载jQuery，通常使用自己服务器上的jQuery，那么用户在首次访问的时候就会下载jQuery文件；而如果我们使用百度CDN上的jQuery，那么只要用户访问过百度，在首次访问自己的网站时也已经存在jQuery而无需下载了 引入百度CDN的jQuery： 如果CDN地址加载失败，则使用自己服务器上的：
